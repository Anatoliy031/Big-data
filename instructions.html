Пошаговая инструкция по загрузке файлов на GitHub

Чтобы опубликовать подготовленные HTML‑страницы в репозитории Anatoliy031/Big-data на GitHub, выполните следующие шаги. Инструкция рассчитана на пользователя без опыта программирования.

Подготовка файлов

  1. Скачайте все файлы, приложенные к ответу: index.html, glossary.html, roadmap.html, templates.html, pricing.html, presentation.html, instructions.html и изображение 1917646a-0382-46ff-9afa-75b04419c005.png. Сохраните их в одну папку на своём компьютере.

Загрузка в репозиторий GitHub

  1. Откройте браузер и перейдите на сайт github.com. Авторизуйтесь под своей учётной записью.
  2. Найдите репозиторий Anatoliy031/Big-data (в меню «Repositories» или по ссылке). Кликните на его название.
  3. На странице репозитория нажмите кнопку “Add file”, затем выберите “Upload files”.
  4. В открывшемся окне перетащите все подготовленные файлы и изображение в область загрузки или нажмите «choose your files» и выберите их вручную.
  5. Убедитесь, что имена файлов совпадают:
    • index.html – главная страница с кнопками.
    • glossary.html – расшифровка сокращений.
    • roadmap.html – дорожная карта внедрения.
    • templates.html – шаблоны договора, SLA и чек‑листов.
    • pricing.html – прейскурант и бандлы.
    • presentation.html – интерактивная карта подстанций.
    • instructions.html – текущая страница с инструкцией.
    • 1917646a-0382-46ff-9afa-75b04419c005.png – изображение для карты.
  6. При необходимости создайте подпапку (например, colocation-presentation) для упорядочивания файлов. Вы можете сделать это в поле «Commit directly» указав путь, например colocation-presentation/index.html. Если структура не важна, оставьте файлы в корне репозитория.
  7. Прокрутите страницу вниз. В поле «Commit changes» введите краткое описание, например «Добавлены страницы презентации услуги колокейшн».
  8. Выберите вариант “Create a new branch for this commit and start a pull request” или “Commit directly to the main branch” в зависимости от ваших политик. Для простоты выберите «Commit directly to main».
  9. Нажмите зелёную кнопку “Commit changes”. Файлы загрузятся в репозиторий.

Публикация на GitHub Pages (опционально)

Если вы хотите, чтобы страница была доступна как веб‑сайт, можно включить GitHub Pages:

  1. В репозитории откройте вкладку Settings (Настройки).
  2. Слева выберите пункт Pages.
  3. В разделе «Source» выберите ветку main и папку / или папку, в которой вы сохранили файлы (например, /colocation-presentation).
  4. Нажмите кнопку Save. Спустя пару минут GitHub сформирует ссылку, по которой ваш сайт будет доступен.
  5. Используйте полученную ссылку, чтобы поделиться страницей с коллегами и клиентами.

Если возникнут трудности, обратитесь к коллеге, имеющему опыт работы с GitHub, или воспользуйтесь официальной справкой GitHub.
