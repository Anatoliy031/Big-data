Шаблоны документов

Ниже — каркасы документов: типовой договор + SLA, регламент доступа и чек-листы. Используются корректные термины: «юнит(ы)» (высота в стойке), «кВт» (мощность), «контрольная точка» (вместо Gate), «оперативная помощь персонала» (вместо Remote Hands), «система автоматизации заявок» (без упоминания CRM).

1) Типовой договор оказания услуг колокейшн + SLA

Договор (каркас)

ДОГОВОР № [____] оказания услуг по размещению оборудования (колокейшн)

г. [Город]                                  «[__]» [месяц] [год] г.

1. ПРЕДМЕТ
1.1. Исполнитель: [Полное наименование] предоставляет Заказчику услугу колокейшн
на площадках (подстанциях, далее — ПС) по адресам Приложения №2, включая:
(a) место в стойке или часть стойки (юниты) [кол-во: __U],
(b) электрическую мощность до [__] кВт, 
(c) поддержание климатических условий, 
(d) доступ по Регламенту доступа (Приложение №3),
(e) кросс-соединения к ОDF (оптический кросс),
(f) сопутствующие сервисы: оперативная помощь персонала, сопровождение работ,
(g) иные услуги по заявкам через систему автоматизации заявок.

1.2. Заказчик размещает и эксплуатирует своё оборудование с соблюдением требований
настоящего договора, SLA (Приложение №1), Регламента доступа (Приложение №3),
техтребований (Приложение №2), правил безопасности и действующего законодательства.

2. ОБЯЗАННОСТИ СТОРОН
2.1. Исполнитель обеспечивает доступность помещения, питание (кВт) в выделенной
точке подключения, температурный режим по SLA, пропускной режим, кросс-соединения.
2.2. Заказчик: использует оборудование, соответствующее Приложению №2; соблюдает
Регламент доступа; исполняет требования законов (включая хранение данных — на стороне
оператора связи, при наличии соответствующего статуса); не соединяет оборудование с
технологическими сетями ПС (АСУ ТП/РЗА), если это не предусмотрено договором.

3. ПОРЯДОК ОКАЗАНИЯ УСЛУГ
3.1. Заявки: через систему автоматизации заявок (доступ, кроссы, оперативная помощь персонала,
сопровождение вне графика). 3.2. Сроки исполнения — по SLA.

4. ПИТАНИЕ И УЧЁТ
4.1. Схема питания: линии A/B (при наличии), автоматы защиты; учёт — по смарт-PDU/счётчикам.
4.2. Энергия сверх включённого лимита — по факту (кВт·ч) согласно тарифам.

5. СТОИМОСТЬ И РАСЧЁТЫ
5.1. Стоимость — по Прейскуранту (Приложение №4) и/или коммерческому предложению.
5.2. Оплата: аванс до [__] числа месяца, сверхнормативные работы — по актам.
5.3. Индексация — не чаще одного раза в год.

6. УРОВНИ СЕРВИСА И ОТВЕТСТВЕННОСТЬ
6.1. SLA — Приложение №1. Ответственность Исполнителя ограничена размерами,
указанными в SLA. Заказчик отвечает за свои данные, программное обеспечение,
выполнение обязательных требований (включая хранение данных), информационную безопасность своего оборудования.

7. ДОСТУП И БЕЗОПАСНОСТЬ
7.1. Доступ — по Регламенту доступа (Приложение №3). Сопровождение обязательно.
7.2. Запрещены любые соединения с технологическими сетями ПС (АСУ ТП/РЗА),
если не предусмотрено проектом/договором.

8. СРОК, РАСТОРЖЕНИЕ, ПРОЧЕЕ
8.1. Срок: [1–3] года с пролонгацией; порядок расторжения; возврат к исходному состоянию.
8.2. Применимое право — законодательство РФ; споры — претензионный порядок [30] дней,
далее суд по месту Исполнителя.

Приложения: №1 SLA; №2 Техтребования/лист подключения; №3 Регламент доступа; №4 Методика и прейскурант.

SLA (фрагмент-шаблон)

  • Доступ в помещение ≥ 99,9% за месяц
  • Питание (на розетке/клемме) ≥ 99,95% за месяц (при линиях A/B или UPS N+1)
  • Температура +18…+27 °C ≥ 99% времени
  • Оперативная помощь персонала старт ≤ 60 минут, 24×7
  • Кросс-соединения (стандарт) ≤ 3 рабочих дней
  • Кросс-соединения (ускоренно) ≤ 24 часов (доп. плата)

Исключения: форс-мажор, работы по безопасности, инциденты вне зоны ответственности Исполнителя и т. п.

2) Регламент доступа (шаблон)

Цель и области применения

Обеспечить безопасный, контролируемый и документируемый доступ представителей Заказчика к телеком-зоне ПС.

Роли и требования

  • Сопровождающий Исполнителя: группа по электробезопасности не ниже III.
  • Представители Заказчика: группа по электробезопасности не ниже II.
  • Виды работ: плановые, аварийные, внеурочные; с/без отключений.

Порядок доступа — шаг за шагом

  1. Заявка: подать в систему автоматизации заявок не позднее чем за 1 рабочий день (план) или по телефону диспетчера (авария).
  2. Согласование: время визита, состав бригады (ФИО, контакты, допуски), перечень работ и зона.
  3. Пропускной режим: оформление пропуска; вход через СКУД; выдача СИЗ при необходимости.
  4. Сопровождение: перемещение только с представителем Исполнителя; соблюдение фото/видео-ограничений.
  5. Работы: выполнение задач согласно заявке/наряду; запрет на доступ к технологическим сетям ПС (АСУ ТП/РЗА).
  6. Завершение: приведение зоны в исходное состояние; подпись журнала работ и акта.
  7. Инциденты: немедленно сообщить диспетчеру; следовать указаниям персонала ПС.

Запрещено

  • Открывать силовые щиты и вмешиваться в АСУ ТП/РЗА.
  • Прокладывать кабели вне разрешённых трасс и нарушать огнезащитные проходки.
  • Использовать нагревательные приборы; оставлять мусор/упаковку.

Формы

  • Журнал доступа (СКУД + бумажный журнал).
  • Наряд/распоряжение на работы.
  • Акт выполненных работ/сдачи-приёмки.

3) Чек-листы монтажа (шаблоны)

Чек-лист «Готовность ПС к размещению»

[ ] Помещение выделено; дверь/замок; таблички.
[ ] Стойки 19″, крепёж, заглушки, кабель-организаторы (до 42 юнитов).
[ ] PDU линии A/B, подписаны линии, автоматы защиты.
[ ] Заземление: стойки/двери/ODF на ГЗШ (проверка мультиметром).
[ ] Климат: +18…+27 °C, аварийная сигнализация.
[ ] СКУД, видеонаблюдение, журнал событий.
[ ] ODF размечен; патч-панели промаркированы.

Чек-лист «Монтаж оборудования Заказчика»

[ ] Оборудование соответствует листу подключения (габариты, масса, потребляемая мощность).
[ ] Установка в стойку по юнитам; крепёж; допустимые радиусы изгиба кабелей.
[ ] Питание подключено (линии A/B при наличии), маркировка шнуров.
[ ] Патч-корды к ODF по лоткам; бирки и схемы.
[ ] Проверены базовые тесты: индикация, загрузка, сетевые интервалы, температурные датчики.
[ ] Оформлены акты, фотофиксация зоны монтажа.

Чек-лист «Сдача/приёмка и ввод в эксплуатацию»

[ ] Серийные номера и инвентарные нанесены/сверены.
[ ] Схема питания, кроссов, портов ODF приложена к акту.
[ ] Учётные данные и контакты 24×7 обменены.
[ ] Доступы заведены; правила заявок в системе автоматизации согласованы.
[ ] Точки измерения (кВт·ч) привязаны; отчётность проверена.
[ ] Зона приведена в исходное состояние; мусор вывезен.

Примечание: при необходимости добавьте отраслевые требования (ПБ, ИБ/КИИ) и внутренние контрольные листы.
